### 2. Groq API Limit Önlemleri — Mevcut Durum ve Riskler
**Mevcut önlemler (iyiler):**
- `_min_interval = 3` saniye istekler arası bekleme.
- `_max_retries = 3`, rate limit hatasında regex ile retry-after çıkarımı.
- `MAX_CHARS_PER_ARTICLE = 300` (~75 token), `MAX_ARTICLES_PER_BATCH = 15`.
- `GROQ_CONTEXT_LIMIT = 6000` karakter limiti.
- Batch'ler arasında `time.sleep(10)` (main.py).
- `response_format={"type": "json_object"}` kullanımı.

**Eksiklikler / Riskler:**
1. `max_tokens=4096` çok yüksek — JSON response için 512-1024 yeterli. Fazla token = daha pahalı, daha yavaş, rate limit'e daha yakın.
2. `temperature=0.05` aşırı düşük — 0.1 daha dengeli sonuç verir.
3. Batch size tutarsızlığı: `main.py` batch_size=10, `analyzer.py` MAX_ARTICLES_PER_BATCH=15. İkisi de 10 olmalı.
4. Fallback model sayısı sadece 2. Groq free tier'da model başına limit ayrıdır, 3. model eklemek güvenliği artırır.
5. `MIN_VALID_EVENT_YEAR = 2025` hardcoded. 2026'ya güncellenmeli veya dinamik hale getirilmeli.
6. Prompt truncation 6000 karakter, gerçek token oranı 4:1 değil prompt yapısına göre değişir. Daha konservatif limit (4000-5000 karakter) daha güvenli.
7. Groq free tier dakikada ~20 request limit. Batch+retry kombinasyonunda bu sınır aşılabilir. Daha agresif bekleme süreleri gerekli.

### 3. Streamlit UI (app.py) Sorunları
- `pydeck as pdk` import edilmiş ama hiç kullanılmıyor — gereksiz import, paket bağımlılığı.
- CSS 170 satır inline, tekrar eden pattern'ler var, düzenlenebilir.
- `extract_source_domain()` içinde Google News domain kontrolü var ama artık URL'ler çözülmüyor, mantık güncellenmeli.
- `st.download_button(type="primary")` — Streamlit <1.28'de hata verebilir, `type` yerine `use_container_width=True` daha güvenli.
- `summary_tr` öncelikli gösterimde Google News makalelerinde boş kalabilir (çünkü sadece title+summary var, LLM çevirisi yapılamayabilir). Fallback mantığı iyileştirilebilir.
- Eski (stale) olayların görsel vurgusu zayıf — grayscale + opacity daha belirgin olabilir.
- Ticker'da `summary_tr` boşsa boş string gösteriliyor, fallback gerekli.

### 4. Genel Kod Kalitesi
- `main.py`'de inline import'lar var (`from analyzer import normalize_incident_type, MIN_VALID_EVENT_YEAR`). Bunlar modül başına taşınmalı.
- `analyzer.py`'de `MIN_VALID_EVENT_YEAR = 2025` hardcoded kalmış, dinamik yapılmalı.
- `collectors.py`'de Google News link çözme kaldırıldı, enrich mantığı güncellendi — iyi.

## Adımlar

### 1. Gereksiz Dosyaları Sil
- `test_folium.py` sil.
- `scripts/fix_existing_dates.py` sil.

### 2. Groq API Limit Optimizasyonu (`scripts/analyzer.py`)
- `max_tokens=4096` → `max_tokens=1024` (JSON çıktı için fazlasıyla yeterli).
- `temperature=0.05` → `temperature=0.1` (daha dengeli).
- `MAX_ARTICLES_PER_BATCH = 15` → `MAX_ARTICLES_PER_BATCH = 10` (main.py ile tutarlı).
- Fallback model listesine 3. model ekle: `llama-3.1-8b-instant` (hızlı, ucuz, son çare).
- `_min_interval = 3` → `_min_interval = 4` (biraz daha konservatif).
- Batch sonrası bekleme: main.py'de `time.sleep(10)` → `time.sleep(15)` (Groq free tier güvenliği).
- `GROQ_CONTEXT_LIMIT = 6000` → `GROQ_CONTEXT_LIMIT = 5000` (daha konservatif, token taşmasını önler).
- `MIN_VALID_EVENT_YEAR = 2025` → dinamik: `datetime.now(timezone.utc).year`.

### 3. main.py Düzenlemeleri
- Inline import'ları kaldır, modül başına taşı.
- Batch sonrası `time.sleep(10)` → `time.sleep(15)`.
- `build_geocode_query` ignore listesine "resort" ekle.

### 4. Streamlit UI Modernizasyonu (`app.py`)
- `pydeck as pdk` import'unu kaldır.
- CSS'i düzenle: gereksiz tekrarları kaldır, daha kompakt hale getir.
- `extract_source_domain()` güncelle: Google News artık çözülmüyor, domain doğrudan Google News kalacak, bu yüzden "Google News" yerine "🔗 News Link" veya benzeri göster.
- `st.download_button(type="primary")` → `st.download_button(use_container_width=True)`.
- Ticker'da `summary_tr` boşsa `summary_en` fallback ekle.
- Stale olay kartlarında grayscale + opacity efekti ekle (`is_stale` kontrolü ile).
- `summary_tr` boş olan makalelerde card içinde `summary_en` göster, ama "🇬🇧 English only" badge ekle.
- "No data" durumunda daha dostane mesaj ve örnek filtre önerileri göster.
